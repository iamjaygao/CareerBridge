  3. KERNEL → Anywhere: Superuser omnipotence (can view any world)
  4. Anywhere → PUBLIC: Always allowed (marketing is public)

                              KEY GUARANTEES
════════════════════════════════════════════════════════════════════════════════

1. PUBLIC WORLD SOVEREIGNTY
   ✓ "/" always renders public landing page
   ✓ NO automatic redirect for authenticated users
   ✓ Users choose when to enter their world

2. LOGIN DISPATCHER EXCLUSIVITY
   ✓ Only ONE place dispatches users to worlds (LoginPage.onSubmit)
   ✓ Uses explicit priority: is_superuser > is_staff > role
   ✓ Predictable, auditable, testable

3. KERNEL IMMUNITY
   ✓ Kernel world BYPASSES all feature flags
   ✓ Kernel world BYPASSES all governance checks
   ✓ Only is_superuser=True can enter
   ✓ Emergency access guaranteed

4. USER CHOICE
   ✓ Authenticated users see "My World" button
   ✓ Clicking "My World" dispatches to their sovereign world
   ✓ No forced navigation away from PUBLIC world
