════════════════════════════════════════════════════════════════════════════════
                           KEY DESIGN PRINCIPLES
════════════════════════════════════════════════════════════════════════════════

1. KERNEL SOVEREIGNTY
   • Kernel world is the root of trust
   • Superusers can ALWAYS access kernel
   • No userland operation can lock out root

2. FEATURE FLAG IMMUNITY
   • Kernel bypasses ALL feature flag checks
   • Even if all features are frozen (OFF), kernel remains accessible
   • Guarantees emergency access

3. CLEAR BOUNDARIES
   • Each world has distinct namespaces and purposes
   • No overlap between staff admin (/admin) and superuser (/superadmin)
   • Userland cannot escalate to kernel

4. AUDIT EVERYTHING
   • All operations tagged with world context
   • Full traceability for compliance and security
   • Can filter logs by world: "Show all kernel operations"

5. DEFENSE IN DEPTH
   • Multiple layers: world → auth → feature flags → permissions
   • Hard gate for kernel (not configurable)
   • Fail-safe: unknown paths default to public world
