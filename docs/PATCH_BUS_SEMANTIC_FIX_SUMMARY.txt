PROBLEM IDENTIFIED:
  Registry was using array format with numeric indices as keys
  → Frontend expected object format with semantic bus IDs as keys

ROOT CAUSE:
  Old registry format (Phase-A.2.B):
    buses: [ { bus: "AI_BUS", status: "FROZEN", ... }, ... ]
  
  Expected format (v0.2):
    buses: { "AI_BUS": { state: "OFF", workloads: [...] }, ... }

Backend (Python):
  BUS_POWER = {
    "KERNEL_CORE_BUS": "ON",
    "AI_BUS": "OFF",
    ...
  }

Frontend (JSON):
  {
    "buses": {
      "KERNEL_CORE_BUS": { "state": "ON", ... },
      "AI_BUS": { "state": "OFF", ... }
    }
  }

This ensures:
✓ Single source of truth (bus IDs)
✓ Type-safe iteration (Record<string, BusEntry>)
✓ O(1) bus lookup by ID
✓ Semantic clarity (no numeric keys)
