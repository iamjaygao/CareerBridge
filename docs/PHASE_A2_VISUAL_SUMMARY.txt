Manual frozen lists inevitably drift as developers freeze/unfreeze modules
without updating the list. This causes 404 noise, confusion, and breaks
Phase-A's zero-noise requirement. Auto-scanning makes the code itself the
source of truth, ensuring the registry stays current without manual
intervention. The static file approach maintains Phase-A's constraint that
SuperAdmin must not trigger legacy runtime or make API calls to frozen
modules. This design is deterministic, testable, and auditable.
